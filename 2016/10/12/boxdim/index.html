盒子模型

盒子模型到底是个什么鬼

形象化展示一个html标签由什么组成，由content padding border margin组成，如下图

有什么问题

存在两种模型：w3c标准盒子模型，ie6以下的怪异盒子模型

w3c标准的盒子模型

外盒尺寸计算（元素空间尺寸）
Element空间高度 = content height + padding + border + margin
Element 空间宽度 = content width + padding + border + margin
内盒尺寸计算（元素大小）
Element Height = content height + padding + border （Height为内容高度）
Element Width = content width + padding + border （Width为内容宽度）

ie的怪异盒子模型

外盒尺寸计算（元素空间尺寸）
Element空间高度 = content Height + margin (Height包含了元素内容宽度，边框宽度，内距宽度)
Element空间宽度 = content Width + margin (Width包含了元素内容宽度、边框宽度、内距宽度)
内盒尺寸计算（元素大小）
Element Height = content Height(Height包含了元素内容宽度，边框宽度，内距宽度)
Element Width = content Width(Width包含了元素内容宽度、边框宽度、内距宽度)

两者的外盒模型尺寸计算一致，内盒尺寸计算不同，ie的怪异模型content width 包括了padding和border在内

实际使用

  1. 设置box-sizing属性为border-box，统一使用ie的怪异模型(ps: 兼容ie8+)

    box-sizing ： content-box(标准模型) || border-box || inherit

  2. 不要给元素有指定宽度高度的元素添加内边距和边框，而是尝试将内边距或外边距或边框添加到元素的父元素和子元素。

  3. 针对有宽高同时有padding或者边框的，将宽高的数值减去内间距以及边框的。

盒子模型在不同情况下的表现

一个块级元素在没有指定宽度的情况下，并且是static或者relative定位，其宽度为100%(相对其最近的父元素)。同时padding和border会向内推动而不是向外扩展

若指定了宽度为100%，则padding和border会向外扩展

无宽度的绝对定位盒子，盒子的宽度跟随盒子里的内容，直到父元素的宽度的100%，若内容是文本会自动换行

块级元素之间的垂直margin，两个块级元素之间的距离不是第一个元素margin-bottom加上第二个元素的margin-top，而是两者之中的最大值，这个现象称为 margin 的“塌陷”现象，即较小的 margin 塌陷到较大的 margin 中。
